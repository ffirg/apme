## Community collection modules

Flags tasks that use modules from `community.*` collections. Community
collections are maintained by the open-source community and are not
covered by Red Hat support. Where possible, use a certified or validated
collection instead.

This rule does **not** flag:

- `ansible.builtin.*` / `ansible.legacy.*` (builtins)
- Certified collections (e.g., `cisco.ios.*`, `amazon.aws.*`)
- Validated collections (e.g., `ansible.posix.*`)
- Short module names that resolve to builtins (those are an M001 concern)

### Example: violation

```yaml
- name: Set sysctl value
  community.general.sysctl:
    name: net.ipv4.ip_forward
    value: "1"
```

### Example: pass

```yaml
- name: Set sysctl value
  ansible.posix.sysctl:
    name: net.ipv4.ip_forward
    value: "1"
```
